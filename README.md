# Disease Prediction using Logistic Regression

Predicts one of 5 conditions (**Healthy, Diabetes, Anemia, Thalassemia, Thrombocytopenia**) from a 24-feature blood panel, using a multinomial logistic regression model. Includes a training notebook and a live Streamlit dashboard.

## Project structure

```
disease_prediction/
├── data/
│   └── Blood_samples_dataset_balanced_2_f_.csv   # source dataset (2351 rows, 24 features, 5 classes)
├── Disease_Prediction_Logistic_Regression.ipynb  # full pipeline: EDA -> training -> evaluation
├── train_pipeline.py                              # same pipeline as a plain script (re-runnable)
├── app.py                                          # Streamlit dashboard (predict + insights + performance)
├── models/                                         # generated by the notebook/script
│   ├── logistic_regression_model.pkl
│   ├── scaler.pkl
│   ├── label_encoder.pkl
│   ├── feature_names.pkl
│   ├── metrics.json
│   └── feature_stats.json
├── assets/                                         # generated charts (used by both notebook & dashboard)
└── requirements.txt
```

## Setup

```bash
pip install -r requirements.txt
```

**Important:** install from `requirements.txt` exactly as given (don't `pip install streamlit scikit-learn` etc. ad hoc) — the versions are pinned to match the environment the model files were pickled with. NumPy in particular changed its internal pickle format in version 2.0, so an older NumPy trying to load these `.pkl` files will fail with a `KeyError` during unpickling. If you already have a Python environment with older packages, either use a fresh virtual environment for this project, or run:

```bash
pip install --upgrade -r requirements.txt
```

If you still hit a pickle/version error after that, the safest fix is to regenerate the artifacts locally with whatever versions you have: just run `python train_pipeline.py` (or re-run the notebook) — it will create fresh `.pkl` files matching your installed package versions.


## 1. Train the model

Either open and run the notebook end-to-end:

```bash
jupyter notebook Disease_Prediction_Logistic_Regression.ipynb
```

or run the equivalent script directly:

```bash
python train_pipeline.py
```

Both produce identical artifacts in `models/` and chart images in `assets/`. The repo already includes a pre-trained run, so you can skip this step and go straight to the dashboard — re-run it only if you change the data or modeling code.

## 2. Launch the dashboard

```bash
streamlit run app.py
```

This opens a browser tab with three views:

- **🔮 Predict** — enter a patient's normalized blood-panel values via sliders (grouped by Blood Count, Metabolic Panel, Lipid Panel, Organ Function, Vitals) and get a live prediction with class-probability breakdown.
- **📊 Data Insights** — class distribution, feature correlation heatmap, and key feature distributions by disease.
- **📈 Model Performance** — accuracy, confusion matrix, ROC curves (one-vs-rest), feature importance, and a full classification report.

## Modeling notes

- Features are min-max normalized in the source CSV (0–1 range); `StandardScaler` is additionally applied before fitting for stable optimization, fit only on the training split to avoid leakage.
- Target encoding via `LabelEncoder`; train/test split is 80/20, stratified by class.
- `LogisticRegression(solver="lbfgs")` is used — scikit-learn handles the multinomial (softmax) case natively with this solver for multi-class targets.
- The model reaches ~100% test accuracy on this dataset because it's a clean, balanced synthetic dataset with well-separated classes. On noisier real-world clinical data, expect lower accuracy — treat this as a property of the dataset, not a guarantee of real-world performance.
